# Notification System using Observer Design Pattern (Java)

## Introduction

In modern applications, users expect **real-time updates** such as notifications, alerts, and messages. This project demonstrates how the **Observer Design Pattern** can be effectively used to manage such interactions in a scalable and loosely coupled way.

---

## Objective

* Implement the **Observer Design Pattern** in Java
* Enable dynamic **subscription and unsubscription** of users
* Demonstrate **loose coupling** between components
* Build an interactive **GUI using Java Swing**

---

## Problem Statement

Design a system where users can **subscribe/unsubscribe dynamically** and automatically receive notifications whenever an event occurs in the system.

---

## Methodology

### Components

* **Subject** → `NotificationSystem`
* **Observer** → `User`

### Working Flow

1. User subscribes to the system
2. System stores the user in a subscriber list
3. Admin sends a notification
4. All subscribed users receive updates instantly
